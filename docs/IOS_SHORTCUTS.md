# iOSショートカット

ひとこと箱 v0.4 以降では、画面を開かずに `POST /api/quick-capture` へ送る入口を用意しています。
ショートカット内に `QUICK_CAPTURE_TOKEN` を入れるため、家族の端末内だけで扱ってください。外部に共有しないでください。

## A. ひと言を残すショートカット

目的: 画面を開かずにひと言だけ保存する。

ショートカットの流れ:

1. 「入力を要求」でテキストを聞く。
2. 「URLの内容を取得」で `https://あなたのドメイン/api/quick-capture` に POST する。
3. ヘッダーに `Content-Type: application/json` を付ける。
4. 本文に下の JSON を送る。
5. 成功したら通知で「残しました」と出す。

送信 JSON:

```json
{
  "token": "QUICK_CAPTURE_TOKEN",
  "text": "入力したテキスト",
  "source": "ios_shortcut"
}
```

使い方候補:

- ホーム画面にショートカットを置く。
- Siri で呼ぶ。
- ウィジェットから呼ぶ。

## B. Safari共有シートから残すショートカット

目的: 検索中、見ているページやURLをひとこと箱に残す。

ショートカットの流れ:

1. ショートカットの詳細で「共有シートに表示」をオンにする。
2. 受け取る入力を URL / テキストにする。
3. 必要なら「入力を要求」で追加メモを1行だけ聞く。
4. 「URLの内容を取得」で `https://あなたのドメイン/api/quick-capture` に POST する。
5. ヘッダーに `Content-Type: application/json` を付ける。
6. 本文に下の JSON を送る。
7. 成功したら通知で「残しました」と出す。

送信 JSON:

```json
{
  "token": "QUICK_CAPTURE_TOKEN",
  "text": "追加メモ",
  "source": "ios_share_sheet",
  "url": "共有URL",
  "title": "共有タイトル"
}
```

URL と title がある場合、ひとこと箱には次のような自然な本文として残します。

```text
あとで見る：ページタイトル https://example.com
メモ：追加メモ
```

## C. 声でひとことを残すショートカット

目的: 画面を開かず、声だけでひとことを残す。

ショートカットの流れ:

1. 「入力を要求」を追加し、入力の種類をテキストにする。
2. iPhone / iPad のキーボードでマイクを押して話す、または Siri からショートカットを呼び出して話す。
3. 「URLの内容を取得」で `https://あなたのドメイン/api/quick-capture` に POST する。
4. ヘッダーに `Content-Type: application/json` を付ける。
5. 本文に下の JSON を送る。
6. 成功したら通知で「残しました」と出す。

送信 JSON:

```json
{
  "token": "QUICK_CAPTURE_TOKEN",
  "text": "音声入力したテキスト",
  "source": "ios_shortcut"
}
```

メモ:

- `QUICK_CAPTURE_TOKEN` はショートカット内に入るため、家族端末内だけで扱ってください。
- token を外部に共有しないでください。
- うまく聞き取れないときは、ホーム画面の入力欄を開いてキーボードのマイク入力を使ってください。

## 注意

- token は家族端末内だけで扱い、外部に共有しないでください。
- URLを残すときは、個人情報やログイン中ページに注意してください。
- 医療、お金、住所、子どもの詳細、職場機密、公的書類写真、他人が写る写真は、家族βではまだ避けてください。

# PRODUCT PRINCIPLES

## このプロダクトの思想

ひとこと箱は、家族の日常に埋もれている改善の種を静かに受け止めるための受け皿です。

ユーザーがすることは、基本的に「ひとこと残す」だけです。整理、分類、優先順位付け、開発チケット化は裏側で扱います。

## 静かな UI の重要性

- ホーム画面は一番よく使う場所なので、判断コストを極力下げる
- 情報を出しすぎると、入力前に疲れる
- 家族向けの空気を守るため、ダッシュボード感や管理画面感を避ける
- `quiet` はモードではなく設計原則として扱う
- 静かさをユーザーに切り替えさせるのではなく、常に静かな表側を保つ

## ユーザーに分類させない理由

- 「正しく書くこと」が目的になると、残す量が減る
- 雑なぼやきや違和感ほど、後から見ると価値がある
- IT 用語やタスク管理の概念に乗らない声も拾いたい
- 共有するか、誰が書いたか、どのカテゴリかを入力時に求めない

## 表の UI と裏の構造化を分ける理由

- 表では入力負荷を下げる
- 裏では AI 活用や自動化に耐えるように型と分析結果を持つ
- 同じデータを、家族向け UI と開発者向け UI の両方で使い回せる
- 家族で使うとしても、最初は共有アプリにしない
- 表側は「入れるだけ」、裏側は review / weekly / developer で静かに整理する

## v0 の設計姿勢

- まずはローカルで確実に動く
- 画像や将来同期を考えて IndexedDB を使う
- AI 連携は後付けできるよう interface を先に切る
- 通常画面では分析っぽさを見せない
- ホームにはメンバー切替、共有設定、設定項目を増やさない
